This document describes manifest for ZeroVM NaCl application.
-------------------------------------------------------------


Description:
The manifest is a text file which must be provided in order to run a NaCl application inside ZeroVM. 
Manifest samples can be found in "samples" directory.


Structure:
- each valid line must contain exactly only one key and one value separated by exactly one '=' sign
- both key and value may contain spaces
- key/value case does matter: "key" and "KeY" are treated as a different keywords
- spaces around key/value will be ignored
- lines are delimited with EOL (unix or windows style)
- invalid lines will be ignored
- lines with unknown keywords (not mentioned below) will be ignored


Keywords:
Input/Output
  Input -- name of the input channel/file
  InputMaxGet -- total number of bytes that can be read from input file
  InputMaxGetCnt -- how many "get" syscall invocations are allowed. n/a for mounted resources
  InputMaxPut -- n/a
  InputMaxPutCnt -- n/a
  InputMode -- 0 - pre-mounted channel, 1 - pre-loaded, 2 - preallocated from network
  Output -- name of the output channel/file
  OutputMaxGet -- total number of bytes that can be read from output file
  OutputMaxGetCnt -- how many "get" syscall invocations are allowed. n/a for mounted resources
  OutputMaxPut -- total number of bytes that can be written to output file
  OutputMaxPutCnt -- how many "put" syscall invocations are allowed. n/a for mounted resources
  OutputMode -- 0 - pre-mounted channel, 1 - pre-loaded, 2 - preallocated from network
  UserLog -- user log file name. gets/puts/e.t.c. are unlimited
  UserMaxLogGet -- n/a
  UserMaxLogGetCnt -- n/a
  UserMaxLogPut -- n/a
  UserMaxLogPutCnt -- n/a
  UserLogMode -- n/a
  NetInput -- reserved
  NetInputMax -- limit for receive bytes
  NetInputMaxGet -- reserved
  NetInputMaxGetCnt -- reserved
  NetInputMaxPut -- reserved
  NetInputMaxPutCnt -- reserved
  NetInputMode -- reserved
  NetOutput -- reserved
  NetOutputMax -- limit for send bytes
  NetOutputMaxGet -- reserved
  NetOutputMaxGetCnt -- reserved
  NetOutputMaxPut -- reserved
  NetOutputMaxPutCnt -- reserved
  NetOutputMode -- reserved

User space
  ContentType -- reserved
  TimeStamp -- time/seed
  XObjectMetaTag -- custom attributes
  UserETag -- checksum of the user output

Report request
  ReportRetCode -- request exit code of ZeroVM executable
  ReportEtag -- request checksum of the user output
  ReportUserRetCode -- request exit code of the user program
  ReportContentType -- reserved
  ReportXObjectMetaTag -- request custom attributes (set by user)

ZeroVM control
  Version -- ZeroVM version
  Log -- ZeroVM log file name
  Report -- report file name
  Nexe -- nexe file name
  NexeMax -- maximum allowed nexe size in bytes
  NexeEtag -- reserved for "fast validation"
  Timeout -- total ZeroVM run time 
  MemMax -- total available memory size for nexe
  CPUMax -- cpu time allotted to nexe execution
  SyscallsMax -- number of syscall invocation allowed to nexe
  SetupCallsMax -- number of setup calls allowed to nexe
  Blob -- blob library (if it will retain)
  CommandLine -- command line arguments for nexe
	Networking -- ZeroVM networking indication flag (temporary)
